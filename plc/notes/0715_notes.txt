===================================
	MC / MCR 마스터컨트롤 set/reset
===================================

use: 회로 구성 변경, if block 같은 느낌; [MC N0 signal] 로 N0 블록을 활성화
ex) 차량에 기능은 넣어야 함; 근데 모든 사람이 슈퍼카 옵션을 넣지 않음; 그럴때 한 블록을 활성화/비활성화

;선택회로;
;수동회로;
;자동회로;
;공통회로;
*필수적으로 구분해야함

N0 << highest priority during race conditions
-> 마지막에 만약 MCR 로 닫는 부분만 있다면 (사이에 회로 X) 그냥 MCR N0 하면 다 닫힘.
-> 이 nesting 은 그냥 if statement 안속으로 nesting 하는 개념과 비슷함

이해가 잘된 지점
- 비트 시프트 / WOR 개념은 이미 전자 과목을 이수하면서 머리에 때려 박혔기 때문에
	어렵지 않음
- 다만, 각각의 bitwise operation 을 수행하는 작업을 코딩 테스트에서 좀 헷갈렸던 기억이 남
다시 정리 :
	value & (1 << k)      // 특정 비트 확인
	value |= 1 << k       // 비트 켜기
	value &= ~(1 << k)    // 비트 끄기
	value ^= 1 << k       // 비트 토글

'프로그래밍'으로 넘어가는 중?
- 전통적인 래더에서 다음 단계로 나아가는 중; 래더 -> 시퀀스 -> ST
- 옛날 어셈블리와 비슷한 call, ret, jump 등 걔념을 적용한 시퀀스 syntax 배움


이해가 안된 지점
- 초반에 LATCH 릴레이 부분; 정전이 나도 상태를 기억하는 래치는 내부 릴레이와 
	무엇이 다른가 ?  
	- 프레스: 손이 프레스 안에 있다 X1 - 1 (Latch) = 있다는 사실을 기억하고 있으니
		전기가 들어와도 바로 손을 찧지 않는다
		- M (내부릴레이) 는 정전과 함께 X1 = 1 -> 0 으로 변하고, 프레스 버튼이 눌려진
			상태라면 정전에서 돌아오는 순간 프레스 사고 위험  
	(좀 간추린 예제긴 하지만, latch 는 전기가 공급 안되도 상태를 유지한다는 개념을 들고가면 될듯)
